# Diligence Packet Export

## Purpose
This document defines how Monyawn staff should assemble and send a premium enterprise diligence packet.

It turns the buyer-facing materials into a repeatable export order so the packet is:
- easy to review
- consistent across deals
- aligned to current product truth
- safe for enterprise and procurement conversations

## Primary Owners
- Diligence Packet / Sales Enablement Lead
- Trust Center / Public Documentation Owner

Supporting owners:
- Customer Assurance / Security Questionnaire Lead
- Repository Steward / Documentation Systems Lead
- CRO / Revenue Lead
- Privacy Lead
- Security Lead

## Core Rule
Send the smallest packet that answers the buyer’s real need.

Do not overload the first send with every internal document. Start with the right packet for the audience and expand only when the buyer’s diligence depth increases.

## Audience-Specific Guidance

| Audience | Start With | Add Next | Avoid Sending First |
|---|---|---|---|
| Executive sponsor | one-pager, buyer packet | trust center | deep operating docs |
| Procurement | buyer packet, trust center | security questionnaire kit | internal redline guidance |
| Security team | security questionnaire kit | security, privacy, AI governance | broad marketing or strategy docs |
| Legal / contracts | buyer packet | redline guidance, controlled review docs | raw engineering or internal-only materials not needed for the issue |
| Technical architecture reviewers | buyer packet | deployment posture docs | speculative future-state architecture |

## Packet Assembly Rules
- Keep file order intentional.
- Keep filenames stable.
- Include only the documents needed for the audience and stage.
- If a document would create unsupported expectations, do not send it without owner review.
- If a question is not answered by the packet, route it rather than padding the packet with loosely related materials.

## Send Checklist
- packet matches the buyer’s audience and stage
- no unsupported hosted, sovereign, or compliance claims are implied
- controlled-review topics were routed before sending
- links and attachments are current
- a staff owner is identified for follow-up
