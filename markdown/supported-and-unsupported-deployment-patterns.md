# Supported And Unsupported Deployment Patterns

## Purpose
This document gives Monyawn a simple technical truth table for deployment-related questions.

## Supported Today
- local-first usage on user-controlled devices
- browser-based workflow use without required company-side opportunity-data retention
- export-first handoff through ZIP packages with JSON restore authority and human-readable derivatives
- customer-managed discussion only where technical, privacy, and legal review confirms fit

## Not Supported As Current Product Offers
- Monyawn-hosted multi-tenant SaaS as a documented live enterprise offer
- region-specific hosted data residency guarantees
- sovereign cloud claims
- isolated enterprise-hosted environments as a documented current offering
- regulated-environment commitments derived only from buyer preference

## Review-Required Exploratory Patterns
- customer-managed internal deployment assumptions
- premium implementation environments
- future hosted or enterprise-managed patterns
- isolated or sovereignty-sensitive patterns

## Engineering Interpretation Rule
Engineering may discuss possible future shapes internally, but public, buyer-facing, or contractual language must stay within documented support boundaries.

## Summary
The current product support line is intentionally narrow. That clarity is part of the premium posture.
