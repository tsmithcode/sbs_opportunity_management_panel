# Resume Tuning Strategy

## Goal
Reframe the supplied resume so SBS sees:
1. hands-on developer fit
2. Autodesk design-tools relevance
3. desktop engineering software strength
4. safe honesty around Revit

## Adjustments Needed
### Heading
Prefer a more execution-oriented heading than pure architect branding.

**Suggested heading options:**
- Software Engineer | Autodesk Design Tools | C# / .NET
- Senior Software Engineer | Autodesk & Engineering Systems
- Software Developer | Autodesk APIs | Desktop Engineering Tools

### Summary
Reduce enterprise breadth slightly. Increase emphasis on:
- C#/.NET implementation
- desktop applications
- Autodesk APIs
- engineering workflows
- performance / correctness / data integrity

### Relevance Ordering
Move the most SBS-relevant proof higher inside bullets:
- Autodesk add-ins
- AutoCAD/Inventor integrations
- desktop engineering tooling
- issue debugging
- deployment/testing discipline
- geometry/model/data correctness language

### Gap Handling
Do not claim Revit work if not evidenced.
Instead say:
- Autodesk platform development across Inventor/AutoCAD/Vault
- fast ramping across adjacent APIs and engineering workflows

## Resume Tone
Target tone should read:
- builder
- reliable
- technical
- domain-aware
- collaborative

Not:
- purely strategic
- purely managerial
- abstract
